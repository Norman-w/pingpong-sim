# 放高球击球理念（领域笔记）

>
> 状态：理念记录（尚未实现）

## 两个合适的击球窗口

对方放高球落到我方台面并弹起后，通常认为有两类合适的击球点：

### 1. 上升初期（弹起后尽快下压）

- 时机：来球在我方台面落台后弹起，进入上升初期。
- 高度条件：只要弹起高度已经**高于球网**，且自身能够得着，就可以向下扣杀/按压。
- 特点：球速相对可控，属于「能跳/能伸到就尽早结束回合」的进攻窗口。

### 2. 下降末期（拖到低位再处理）

- 时机：弹起过最高点后，进入下降末期。
- 高度条件：只要自己够得到，**几乎任意高度都可以处理**——甚至低到接近地面。
- 特点：给反应与移动留时间，但触球瞬间球的竖直下落速度更大，对预判要求更高。

```text
落台 → 上升初期(可下压) → 最高点 → 下降期 → 下降末期(可低位处理) → 再落台/坠地
         ▲ 窗口 A                              ▲ 窗口 B
```

## 小孩子 / 近网高吊的特殊困难

对小孩子，尤其是**近网放高球**：

1. **起跳点（上升窗口）往往用不了**  
   身高、起跳、上步与力量不足时，弹起后高于球网且自己够得着的「按下去」窗口可能根本不存在或极短。

2. **只能被迫拉下降点**  
   于是只能等球过最高点后下落到自己够得着的高度再打。

3. **下降段并不「好打」**  
   - 从最高点到「小孩子够得着的低点」之间，有一段很长的近似自由落体时间。  
   - 球越往下落，竖直速度越大。  
   - 越想在**更低、更适合小孩子的高度**回球，球就越快，越难「拉到」。  
   - 这里先不谈能否拉上台；**仅「拍碰到球」本身**就已经要求对自由落体节奏有很准的预判。  
   - 若不主动把球「拖回去」或改变回球策略，低位处理难度会进一步上升。

直觉结论：

> 对成人进攻手，「上升高点下压」与「下降低位处理」都是合法且常见的选择；  
> 对儿童近网高吊，「上升窗口」常不可达，而「下降窗口」虽可达，却因下落加速而更难把握。

## 拟引入的模拟概念（待设计）

基于以上理念，后续可在教学/跟球模拟中引入两个相关概念（先记名，后建模）：

### 1. 落台后位点的可处理能力（Processability）

对来球在我方台面落台后的轨迹上每一个（或若干采样）位点，评价：

- 当前高度是否够得着（与接球方身高、起跳、站位、伸拍范围相关）
- 当前竖直/水平速度是否在可预判、可挥拍时间窗内
- 是否仍满足规则（已落台、未二次落台等）
- 综合得到该点的「可处理难度 / 可处理分数」

### 2. 该点的可处理技术集合（Processable Technique Set）

同一位点，不同技术的可行性不同。例如：

| 轨迹区段 | 可能技术集合（示意） | 备注 |
| --- | --- | --- |
| 上升初期、高于球网、够得着 | 扣杀、快攻、高点击打 | 儿童近网常为空集或极难 |
| 最高点附近 | 扣杀 / 主动进攻（视够球能力） | 取决于伸拍与起跳 |
| 下降中后段、中等高度 | 拉球、带打、过渡 | 预判要求升高 |
| 下降末期、很低甚至近地 | 捞球、托带、被动过渡 | 「碰到」难于「上台」 |
| 任意区段但人够不着 | 空集 | 只能等下一窗口或失分 |

同一球路对不同接球方（成人俱乐部 / 儿童）应得到**不同的可处理曲线与技术集合**。

## 与当前模拟的差距（备忘）

当前实现更接近「单一建议击球点 + 是否犯规（如扣杀须先落台）」。

尚未表达：

- 上升窗口 vs 下降窗口的双峰可处理结构
- 儿童因够球能力不足导致上升窗口坍塌
- 下降越低、球越快、命中预判越难的连续难度变化
- 「该点能用哪些技术」而不只是「推荐一个点」
